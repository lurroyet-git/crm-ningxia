# 宁夏CRM作战地图 — 管理员手册

> 本文档面向系统管理员，详细介绍用户管理、权限配置、系统维护等管理操作。

---

## 一、用户管理

### 1.1 添加用户

**操作路径**：系统管理 → 用户管理 → 添加用户

**操作步骤**：

1. 点击【添加用户】按钮
2. 填写用户基本信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 用户名 | 是 | 登录账号，建议使用姓名拼音，如 zhangwei |
| 姓名 | 是 | 真实姓名，如 张伟 |
| 邮箱 | 是 | 有效邮箱地址，用于密码重置和通知 |
| 手机号 | 否 | 建议填写，用于紧急联系 |
| 部门 | 否 | 所属部门 |
| 角色 | 是 | 选择预设角色（见角色权限章节） |
| 初始密码 | 是 | 系统生成或手动设置，首次登录需修改 |

3. 点击【保存】
4. 系统会自动发送激活邮件（如配置了SMTP）

> 用户名一旦创建不可修改，请谨慎录入。

[截图: 添加用户页面]

### 1.2 编辑用户

1. 进入【用户管理】页面
2. 在用户列表中找到目标用户
3. 点击【编辑】按钮
4. 修改需要更新的字段（用户名不可修改）
5. 点击【保存】

**可编辑字段**：

| 字段 | 是否可编辑 | 备注 |
|------|------------|------|
| 姓名 | 是 | — |
| 邮箱 | 是 | 需验证新邮箱 |
| 手机号 | 是 | — |
| 部门 | 是 | — |
| 角色 | 是 | 会立即生效 |
| 状态 | 是 | 启用/禁用 |
| 头像 | 是 | 用户也可自行修改 |

### 1.3 禁用用户

**场景**：员工离职、账号异常、临时冻结

**操作步骤**：

1. 进入【用户管理】页面
2. 找到目标用户
3. 点击【禁用】按钮
4. 确认禁用操作

**禁用效果**：

- 用户立即无法登录系统
- 已登录会话会被强制踢出
- 用户数据不会被删除
- 历史操作记录保留

> 禁用用户可恢复，删除用户为不可逆操作。建议优先使用禁用而非删除。

### 1.4 启用用户

1. 在用户列表中找到已禁用的用户（状态显示为"禁用"）
2. 点击【启用】按钮
3. 用户可恢复正常使用

### 1.5 重置密码

**场景**：用户忘记密码、账号安全事件

**操作步骤**：

1. 进入【用户管理】页面
2. 找到目标用户
3. 点击【重置密码】按钮
4. 选择重置方式：

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 随机密码 | 系统生成随机密码，发送邮件 | 忘记密码 |
| 指定密码 | 管理员手动设置新密码 | 安全事件 |
| 强制修改 | 用户下次登录强制修改 | 定期更新 |

5. 确认重置操作

> 重置密码后，建议通知用户及时修改为自己的密码。

## 二、角色权限配置

### 2.1 角色定义

系统预设以下角色，也可根据业务需求自定义：

| 角色 | 数据权限 | 功能权限 | 典型用户 |
|------|----------|----------|----------|
| **超级管理员** | 全部数据 | 全部功能 | 系统运维负责人 |
| **管理员** | 全部数据 | 除系统配置外的全部功能 | 业务管理员 |
| **销售经理** | 本部门+下属 | 客户、商机、报表、知识 | 销售主管 |
| **销售代表** | 仅自己 | 客户、商机、拜访 | 一线销售 |
| **项目经理** | 负责的项目 | 项目、客户、运维 | 项目主管 |
| **运维工程师** | 负责的工单 | 运维工单、资产 | 运维人员 |
| **普通用户** | 公开数据 | 查看+部分操作 | 新员工/访客 |
| **只读用户** | 公开数据 | 仅查看 | 审计/高管 |

### 2.2 菜单权限配置

**操作路径**：系统管理 → 角色权限 → 菜单权限

**配置方法**：

1. 选择需要配置的角色
2. 在权限树中勾选该角色可访问的菜单：

| 菜单权限 | 说明 |
|----------|------|
| 可见 | 角色可在左侧菜单看到该入口 |
| 不可见 | 角色看不到该菜单 |

3. 点击【保存】

**权限树结构**：

```
├─ 作战台
│   ├─ 今日概览
│   ├─ 经营报表
│   ├─ 数据导出
│   └─ 团队设置
├─ 项目交付
│   ├─ 项目概览
│   ├─ 项目节点
│   ├─ 会议管理
│   └─ 交付看板
├─ 客户资产
│   ├─ 客户档案
│   ├─ 区域地图
│   └─ 关系网络
├─ 运维管理
│   ├─ 运维工单
│   ├─ 巡检计划
│   └─ 资产台账
├─ 商机营销
│   ├─ 商机池
│   ├─ 跟进记录
│   └─ 拜访计划
├─ 知识分享
│   ├─ 知识素材
│   └─ 培训计划
└─ 系统管理
    ├─ 用户管理
    ├─ 角色权限
    ├─ 系统配置
    ├─ 数据质量
    ├─ 变更检测
    └─ 日志审计
```

[截图: 菜单权限配置页面]

### 2.3 按钮权限配置

**操作路径**：系统管理 → 角色权限 → 按钮权限

**配置方法**：

1. 选择角色和页面
2. 勾选该角色在页面中可执行的操作：

| 按钮权限 | 说明 | 适用页面 |
|----------|------|----------|
| 新增 | 允许创建数据 | 列表页 |
| 编辑 | 允许修改数据 | 列表页/详情页 |
| 删除 | 允许删除数据 | 列表页/详情页 |
| 导出 | 允许导出数据 | 列表页/报表页 |
| 导入 | 允许批量导入 | 列表页 |
| 审核 | 允许审核操作 | 工单/变更页 |

3. 点击【保存】

> **注意**：按钮权限依赖于菜单权限。如果用户没有页面访问权限，按钮权限不生效。

### 2.4 数据权限规则

| 数据权限 | 可见范围 | 说明 |
|----------|----------|------|
| 全部数据 | 系统中所有数据 | 超级管理员、管理员 |
| 本部门及下属 | 本部门+所有子部门数据 | 部门经理 |
| 仅自己 | 自己创建/负责的数据 | 普通员工 |
| 公开数据 | 标记为公开的数据 | 只读用户 |

**数据权限配置示例**：

| 角色 | 客户数据 | 项目数据 | 工单数据 | 商机数据 |
|------|----------|----------|----------|----------|
| 超级管理员 | 全部 | 全部 | 全部 | 全部 |
| 销售经理 | 本部门 | 本部门 | 本部门 | 本部门 |
| 销售代表 | 仅自己 | 仅自己 | 仅自己 | 仅自己 |
| 项目经理 | 负责项目的客户 | 负责的项目 | 项目关联工单 | 无 |
| 运维工程师 | 工单关联客户 | 无 | 分配的工单 | 无 |

## 五、数据质量检查

### 5.1 手动触发检查

**操作路径**：系统管理 → 数据质量 → 质量检查

**检查项目**：

| 检查项 | 规则 | 权重 |
|--------|------|------|
| 客户名称完整性 | 客户名称不能为空 | 高 |
| 客户区域有效性 | 区域必须在标准列表中 | 高 |
| 联系人信息完整性 | 至少一个联系人 | 中 |
| 电话号码格式 | 符合中国大陆电话号码格式 | 中 |
| 邮箱格式 | 符合邮箱格式规范 | 低 |
| 项目日期合理性 | 结束日期不早于开始日期 | 高 |
| 负责人有效性 | 负责人必须是系统用户 | 高 |
| 重复客户检测 | 客户名称相似度>80% | 中 |
| 空值检测 | 必填字段无空值 | 高 |
| 关联完整性 | 外键关联记录存在 | 高 |

**操作步骤**：

1. 点击【开始检查】
2. 系统执行全量数据扫描
3. 查看检查结果报告
4. 查看问题数据列表
5. 导出问题数据（如有权限）

### 5.2 单个客户质量评分

1. 进入【客户资产】→【客户档案】
2. 找到目标客户
3. 点击【质量评分】按钮
4. 查看该客户数据的完整性评分

| 评分等级 | 分数 | 说明 |
|----------|------|------|
| 优秀 | 90-100 | 数据完整，无问题 |
| 良好 | 70-89 | 数据基本完整，少量问题 |
| 一般 | 50-69 | 数据缺失较多，需补充 |
| 较差 | 0-49 | 数据严重不足，需重新录入 |

## 六、变更处理流程

### 6.1 变更检测

系统自动检测以下变更：

| 变更类型 | 检测方式 | 示例 |
|----------|----------|------|
| 数据新增 | 对比数据快照 | 新客户录入、新项目创建 |
| 数据修改 | 对比字段变更 | 客户状态变更、负责人变更 |
| 数据删除 | 标记删除记录 | 客户删除、项目删除 |
| 配置变更 | 配置版本对比 | 权限变更、系统设置变更 |

### 6.2 变更确认流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  变更检测  │───▶│  变更列表  │───▶│  人工确认  │───▶│  归档记录  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                     │
                                     ▼
                               ┌──────────┐
                               │  异常处理  │
                               └──────────┘
```

**操作步骤**：

1. 进入【系统管理】→【变更检测】
2. 查看未确认的变更列表
3. 点击【查看详情】了解变更内容
4. 操作：
   - 【确认】：标记为正常变更
   - 【忽略】：标记为无需关注
   - 【标记异常】：标记为异常变更，需进一步处理

> 异常变更可能指示数据误操作或安全问题，请仔细审查。

### 6.3 手动创建变更记录

1. 进入【系统管理】→【变更检测】
2. 点击【手动创建变更】
3. 填写变更信息：
   - 变更类型
   - 变更对象
   - 变更描述
   - 变更原因
4. 点击【提交】

## 七、通知管理

### 7.1 通知发送

**操作路径**：系统管理 → 通知管理 → 发送通知

| 通知类型 | 说明 | 使用场景 |
|----------|------|----------|
| 系统公告 | 全体用户可见 | 系统升级、功能更新 |

| 待办提醒 | 关联业务对象 | 工单到期、节点提醒 |

**发送步骤**：

1. 选择通知类型
2. 选择接收人（全体/指定用户/指定角色）
3. 填写通知标题和内容
4. 选择发送方式（站内/邮件/企微）
5. 点击【发送】

### 7.2 通知历史

| 字段 | 说明 |

| 发送时间 | 通知发送时间 |

| 接收人 | 通知目标用户 |

### 7.3 通知模板管理

预设通知模板：

| 模板名称 | 使用场景 | 变量 |
|----------|----------|------|
| 密码重置 | 重置密码后发送 | 用户名、新密码、登录地址 |
| 账号激活 | 新用户创建后发送 | 用户名、初始密码、登录地址 |
| 工单指派 | 工单分配后发送 | 工单编号、工单类型、优先级 |
| 节点提醒 | 项目节点到期前发送 | 项目名称、节点名称、到期日期 |
| 拜访提醒 | 拜访计划前发送 | 客户名称、拜访时间、拜访目的 |
| 系统公告 | 系统维护通知 | 维护时间、影响范围、预计恢复 |

## 八、日志审计

### 8.1 操作日志

**操作路径**：系统管理 → 日志审计 → 操作日志

记录所有用户的操作行为：

| 字段 | 说明 |
|------|------|
| 时间 | 操作发生的精确时间 |
| 用户 | 操作人账号和姓名 |
| IP地址 | 操作来源IP |
| 操作模块 | 功能模块 |
| 操作类型 | 新增/编辑/删除/查询/导出/登录/退出 |
| 操作对象 | 被操作的数据ID |
| 操作结果 | 成功/失败 |
| 请求参数 | 操作请求内容（脱敏） |
| 响应信息 | 系统返回信息 |

### 8.2 登录日志

记录所有登录行为：

| 字段 | 说明 |
|------|------|
| 时间 | 登录时间 |
| 用户 | 登录账号 |
| IP地址 | 登录来源IP |
| 登录方式 | 账号密码/SSO |
| 登录结果 | 成功/失败 |
| 失败原因 | 密码错误/账号禁用/账号不存在 |
| 用户代理 | 浏览器/设备信息 |

### 8.3 日志查询与导出

**查询条件**：

| 条件 | 说明 |
|------|------|
| 时间范围 | 指定起始和结束时间 |
| 用户 | 筛选特定用户的操作 |
| 模块 | 筛选特定功能模块 |
| 操作类型 | 筛选新增/编辑/删除等 |
| IP地址 | 筛选特定来源IP |

**导出日志**：

1. 设置查询条件
2. 点击【导出】按钮
3. 选择导出格式（Excel/CSV）
4. 确认导出

> 日志导出权限仅限超级管理员，导出操作本身也会被记录。

### 8.4 日志保留策略

| 日志类型 | 本地保留 | 归档存储 | 清理策略 |
|----------|----------|----------|----------|
| 操作日志 | 90天 | 1年 | 自动清理 |
| 登录日志 | 90天 | 1年 | 自动清理 |
| 错误日志 | 30天 | 90天 | 自动清理 |
| 审计日志 | 1年 | 3年 | 手动归档 |

> 审计日志涉及合规要求，保留时间更长，建议定期归档到独立存储。

## 九、管理员检查清单

### 9.1 每日检查

- [ ] 检查系统健康状态（首页概览）
- [ ] 查看是否有异常告警
- [ ] 检查登录日志是否有异常登录
- [ ] 处理用户账号相关请求

### 9.2 每周检查

- [ ] 审查用户权限变更记录
- [ ] 检查数据导出日志
- [ ] 审查变更检测中的异常标记
- [ ] 检查系统存储空间
- [ ] 备份状态确认

### 9.3 每月检查

- [ ] 用户账号清理（禁用长期未登录账号）
- [ ] 权限审计（确认角色权限合理性）
- [ ] 数据质量检查报告审查
- [ ] 日志归档和清理
- [ ] 系统配置审查

### 9.4 每季度检查

- [ ] 全量数据质量检查
- [ ] 安全策略审查（密码策略、锁定策略）
- [ ] 角色和权限全面审查
- [ ] 备份恢复演练
- [ ] 系统性能评估

---

> 管理员是系统安全的第一责任人，请严格遵守操作规范，定期审查系统状态，确保数据安全和业务连续性。
