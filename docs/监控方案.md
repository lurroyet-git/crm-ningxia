# 宁夏CRM作战地图 — 监控方案

> 本文档定义系统的监控目标、技术选型、告警规则和Dashboard设计，确保运维团队能够实时掌握系统健康状态。

## 一、监控目标

### 1.1 监控维度

| 维度 | 指标 | 目标值 | 监控频率 |
|------|------|--------|----------|
| **可用性** | 系统在线率 | ≥ 99.9% | 每30秒 |
| **可用性** | 接口成功率 | ≥ 99.5% | 每30秒 |
| **性能** | API 平均响应时间 | < 500ms | 每30秒 |
| **性能** | 数据库查询时间 | < 100ms | 每60秒 |
| **性能** | 前端页面加载时间 | < 3s | 每5分钟 |
| **错误率** | HTTP 5xx 错误率 | < 0.1% | 每30秒 |
| **错误率** | 应用异常日志数 | < 10/小时 | 每60秒 |
| **资源** | CPU 使用率 | < 70% | 每30秒 |
| **资源** | 内存使用率 | < 80% | 每30秒 |
| **资源** | 磁盘使用率 | < 85% | 每300秒 |
| **资源** | 网络带宽 | < 80% | 每60秒 |
| **业务** | 日活跃用户数 | 正常波动 | 每日 |
| **业务** | 数据库连接数 | < 80% 最大连接数 | 每30秒 |

## 二、日志收集（Loki）

### 2.1 Loki 架构

| 组件 | 用途 | 部署方式 |
|------|------|----------|
| Loki | 日志聚合存储 | Docker 容器 |
| Promtail | 日志采集器 | Docker 容器或宿主机 |
| Grafana | 日志查询和展示 | 与监控共用 |

### 2.2 Promtail 配置

创建 `/opt/crm-platform/promtail-config.yml`：

```yaml
server:
  http_listen_port: 9080
  grpc_listen_port: 0

positions:
  filename: /tmp/positions.yaml

clients:
  - url: http://loki:3100/loki/api/v1/push

scrape_configs:
  - job_name: crm-backend
    static_configs:
      - targets:
          - localhost
        labels:
          job: crm-backend
          __path__: /var/log/crm/backend/*.log

  - job_name: crm-nginx
    static_configs:
      - targets:
          - localhost
        labels:
          job: crm-nginx
          __path__: /var/log/nginx/*.log

  - job_name: docker-containers
    docker_sd_configs:
      - host: unix:///var/run/docker.sock
        refresh_interval: 5s
    relabel_configs:
      - source_labels: ['__meta_docker_container_name']
        regex: '/(.*)'
        target_label: 'container'
```

### 2.3 日志标签设计

| 标签 | 说明 | 示例值 |
|------|------|--------|
| `job` | 日志来源 | `crm-backend`, `crm-nginx` |
| `level` | 日志级别 | `error`, `warn`, `info` |
| `module` | 业务模块 | `auth`, `project`, `customer` |
| `user_id` | 操作用户 | `user-123` |
| `trace_id` | 请求追踪ID | `trace-abc123` |

### 2.4 常用日志查询

```bash
# 查询所有错误日志
{job="crm-backend"} |= "ERROR"

# 查询特定模块的错误
{job="crm-backend"} |= "ERROR" | json | module="project"

# 查询特定用户的操作日志
{job="crm-backend"} | json | user_id="zhangwei"

# 查询响应时间超过1秒的请求
{job="crm-nginx"} |~ "HTTP/1.1" (5..) | pattern `<ip> - - <_> "<method> <uri> <proto>" <status> <size> <_> <rt>` | rt > 1
```

### 3.2 监控容器配置（docker-compose.monitoring.yml）

```yaml
version: '3.8'

services:
  prometheus:
    image: prom/prometheus:latest
    container_name: crm-prometheus
    restart: always
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
      - ./prometheus-rules:/etc/prometheus/rules

      - "127.0.0.1:9090:9090"
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
      - '--storage.tsdb.retention.time=30d'
    networks:
      - crm-network

  grafana:
    image: grafana/grafana:latest
    container_name: crm-grafana
    restart: always
    environment:
      - GF_SECURITY_ADMIN_USER=admin
      - GF_SECURITY_ADMIN_PASSWORD=YOUR_GRAFANA_PASSWORD
    volumes:
      - grafana_data:/var/lib/grafana
      - ./grafana-dashboards:/etc/grafana/provisioning/dashboards
    ports:
      - "127.0.0.1:3000:3000"
    networks:
      - crm-network

  loki:
    image: grafana/loki:latest
    container_name: crm-loki
    restart: always
    volumes:
      - loki_data:/loki
    ports:
      - "127.0.0.1:3100:3100"
    networks:
      - crm-network

  alertmanager:
    image: prom/alertmanager:latest
    container_name: crm-alertmanager
    restart: always
    volumes:
      - ./alertmanager.yml:/etc/alertmanager/alertmanager.yml
    ports:
      - "127.0.0.1:9093:9093"
    networks:
      - crm-network

  node-exporter:
    image: prom/node-exporter:latest
    container_name: crm-node-exporter
    restart: always
    volumes:
      - /proc:/host/proc:ro
      - /sys:/host/sys:ro
      - /:/rootfs:ro
    command:
      - '--path.procfs=/host/proc'
      - '--path.rootfs=/rootfs'
      - '--path.sysfs=/host/sys'
      - '--collector.filesystem.mount-points-exclude=^/(sys|proc|dev|host|etc)($$|/)'
    networks:
      - crm-network

  postgres-exporter:
    image: prometheuscommunity/postgres-exporter:latest
    container_name: crm-postgres-exporter
    restart: always
    environment:
      - DATA_SOURCE_NAME=postgresql://crm_user:PASSWORD@postgres:5432/crm_production?sslmode=disable
    networks:
      - crm-network

  redis-exporter:
    image: oliver006/redis_exporter:latest
    container_name: crm-redis-exporter
    restart: always
    environment:
      - REDIS_ADDR=redis://redis:6379
      - REDIS_PASSWORD=YOUR_REDIS_PASSWORD
    networks:
      - crm-network

volumes:
  prometheus_data:
  grafana_data:
  loki_data:

networks:
  crm-network:
    external: true
```

### 3.3 启动监控栈

```bash

sudo docker compose -f docker-compose.monitoring.yml up -d

# 验证各组件状态
sudo docker compose -f docker-compose.monitoring.yml ps
```

## 四、告警规则

### 4.1 告警规则文件

创建 `/opt/crm-platform/prometheus-rules/crm-alerts.yml`：

```yaml
groups:
  - name: crm-availability
    rules:
      - alert: CRMServiceDown
        expr: up{job="crm-backend"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "CRM后端服务不可达"
          description: "CRM后端服务已连续1分钟无响应，请立即检查"

      - alert: CRMHighErrorRate
        expr: rate(nginx_http_requests_total{status=~"5.."}[5m]) > 0.01
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "CRM错误率过高"
          description: "5xx错误率超过1%，持续2分钟"

  - name: crm-performance
    rules:
      - alert: CRMApiSlowResponse
        expr: histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m])) > 1
        for: 3m
        labels:
          severity: warning
        annotations:
          summary: "API响应时间超过1秒"
          description: "95%请求响应时间超过1秒，持续3分钟"

      - alert: CRMDatabaseSlowQuery
        expr: rate(pg_stat_activity_max_tx_duration[5m]) > 30
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "数据库慢查询"
          description: "数据库存在超过30秒的长事务"

  - name: crm-resources
    rules:
      - alert: CRMHighMemoryUsage
        expr: (node_memory_MemTotal_bytes - node_memory_MemAvailable_bytes) / node_memory_MemTotal_bytes > 0.8
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "服务器内存使用率超过80%"
          description: "内存使用率: {{ $value | humanizePercentage }}"

      - alert: CRMHighDiskUsage
        expr: (node_filesystem_avail_bytes / node_filesystem_size_bytes) < 0.15
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "服务器磁盘使用率超过85%"
          description: "磁盘使用率: {{ $value | humanizePercentage }}"

      - alert: CRMHighCPUUsage
        expr: 100 - (avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 70
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "CPU使用率超过70%"
          description: "CPU使用率: {{ $value }}%"

  - name: crm-database
    rules:
      - alert: CRMDatabaseDown
        expr: up{job="postgres-exporter"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "PostgreSQL数据库不可达"
          description: "数据库连接失败，请立即检查"

      - alert: CRMDatabaseConnectionsHigh
        expr: pg_stat_activity_count / pg_settings_max_connections * 100 > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "数据库连接数过多"
          description: "连接数使用率: {{ $value }}%"

      - alert: CRMRedisDown
        expr: up{job="redis-exporter"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "Redis缓存服务不可达"
          description: "Redis连接失败，请立即检查"
```

### 4.2 告警规则汇总表

| 告警名称 | 触发条件 | 持续时间 | 严重级别 | 响应时间要求 |
|----------|----------|----------|----------|--------------|
| CRMServiceDown | 后端服务不可达 | 1分钟 | **Critical** | 15分钟 |
| CRMHighErrorRate | 5xx错误率 > 1% | 2分钟 | **Critical** | 15分钟 |
| CRMApiSlowResponse | 95%请求 > 1秒 | 3分钟 | Warning | 1小时 |
| CRMDatabaseSlowQuery | 长事务 > 30秒 | 5分钟 | Warning | 1小时 |
| CRMHighMemoryUsage | 内存使用率 > 80% | 5分钟 | Warning | 4小时 |
| CRMHighDiskUsage | 磁盘使用率 > 85% | 5分钟 | Warning | 4小时 |
| CRMHighCPUUsage | CPU使用率 > 70% | 10分钟 | Warning | 4小时 |
| CRMDatabaseDown | 数据库不可达 | 1分钟 | **Critical** | 15分钟 |
| CRMDatabaseConnectionsHigh | 连接数 > 80% | 5分钟 | Warning | 1小时 |
| CRMRedisDown | Redis不可达 | 1分钟 | **Critical** | 15分钟 |

## 五、告警渠道配置

### 5.1 Alertmanager 配置

创建 `/opt/crm-platform/alertmanager.yml`：

```yaml
global:
  smtp_smarthost: 'smtp.example.com:587'
  smtp_from: 'alert@example.com'
  smtp_auth_username: 'alert@example.com'
  smtp_auth_password: 'YOUR_SMTP_PASSWORD'

route:
  receiver: 'default-receiver'
  group_by: ['alertname', 'severity']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h
  routes:
    - match:
        severity: critical
      receiver: 'critical-receiver'
      continue: true
    - match:
        severity: warning
      receiver: 'warning-receiver'

receivers:
  - name: 'default-receiver'
    email_configs:
      - to: 'ops@example.com'

  - name: 'critical-receiver'
    email_configs:
      - to: 'ops@example.com,manager@example.com'
    webhook_configs:
      - url: 'https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_WECHAT_KEY'
        send_resolved: true

  - name: 'warning-receiver'
    email_configs:
      - to: 'ops@example.com'
    webhook_configs:
      - url: 'https://oapi.dingtalk.com/robot/send?access_token=YOUR_DINGTALK_TOKEN'
        send_resolved: true
```

### 5.2 告警渠道对比

| 渠道 | 适用级别 | 延迟 | 优点 | 缺点 |
|------|----------|------|------|------|
| 企业微信 | Critical + Warning | < 10s | 即时触达、支持@提醒 | 需维护Webhook密钥 |
| 钉钉 | Warning | < 10s | 同企业微信 | 需维护Token |
| 邮件 | 所有级别 | 1-5min | 完整记录、可追踪 | 延迟较大 |
| 短信 | Critical only | < 30s | 强提醒 | 成本较高 |

### 6.3 关键面板指标

#### 系统概览面板

| 面板 | 指标 | 可视化 | 阈值 |
|------|------|--------|------|
| 在线率 | `up{job="crm-backend"}` | Stat / 大数字 | 绿灯: 1, 红灯: 0 |
| API成功率 | `rate(http_requests_total{status!~"5.."}[5m]) / rate(http_requests_total[5m])` | Gauge | 绿灯: >99%, 红灯: <95% |
| 平均响应时间 | `histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))` | Stat | 绿灯: <500ms, 黄灯: <1s, 红灯: >1s |
| 错误率 | `rate(http_requests_total{status=~"5.."}[5m])` | Stat | 绿灯: <0.1%, 黄灯: <1%, 红灯: >1% |
| 响应时间趋势 | `histogram_quantile` P50/P95/P99 | 折线图 | 无 |
| 请求吞吐量 | `rate(http_requests_total[5m])` | 折线图 | 无 |
| 慢接口排名 | 按 `http_request_duration_seconds` 排序 | 表格 | 无 |
| 活跃告警 | `ALERTS` | 表格 | 无 |

#### 数据库监控面板

| 面板 | 指标 | 可视化 |
|------|------|--------|
| 连接数 | `pg_stat_activity_count` | 时间序列 |
| 查询吞吐量 | `pg_stat_database_xact_commit` | 折线图 |
| 慢查询数 | `pg_stat_statements` | 表格 |
| 缓存命中率 | `pg_stat_database_blks_hit / (blks_hit + blks_read)` | Gauge |
| 表大小 | `pg_database_size` | 柱状图 |
| 锁等待 | `pg_locks_wait` | 表格 |

#### 基础设施面板

| 面板 | 指标 | 可视化 |
|------|------|--------|
| CPU使用率 | `100 - rate(node_cpu_seconds_total{mode="idle"}[5m]) * 100` | 折线图 |
| 内存使用率 | `(MemTotal - MemAvailable) / MemTotal * 100` | 折线图 |
| 磁盘使用率 | `(1 - filesystem_avail / filesystem_size) * 100` | 折线图/饼图 |
| 网络流量 | `node_network_receive_bytes_total` / `transmit_bytes_total` | 折线图 |
| 磁盘I/O | `node_disk_io_time_seconds_total` | 折线图 |
| 进程数 | `node_processes_state` | 折线图 |

- 默认账号：`admin`
- 默认密码：在 `.env` 中设置的 `GF_SECURITY_ADMIN_PASSWORD`

> 生产环境强烈建议修改默认管理员密码，并配置LDAP或OAuth认证。
