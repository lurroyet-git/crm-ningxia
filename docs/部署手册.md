# 宁夏CRM作战地图 — 部署手册

> 本文档面向运维工程师，指导生产环境的完整部署流程。

---

## 一、服务器环境要求

### 1.1 硬件配置

| 组件 | 最低配置 | 推荐配置 | 说明 |
|------|----------|----------|------|
| CPU | 4核 | 8核 | 后端服务+数据库+缓存共用 |
| 内存 | 8 GB | 16 GB | Redis和PostgreSQL占用较大 |
| 系统盘 | 100 GB SSD | 200 GB SSD | 操作系统+Docker镜像+日志 |
| 数据盘 | 200 GB SSD | 500 GB SSD | PostgreSQL数据+MinIO文件存储 |
| 网络带宽 | 10 Mbps | 50 Mbps | 文件上传下载场景 |

### 1.2 操作系统要求

| 系统 | 版本 | 状态 |
|------|------|------|
| Ubuntu Server | 22.04 LTS / 24.04 LTS | **推荐** |
| CentOS Stream | 9 | 支持 |
| Debian | 12 (bookworm) | 支持 |
| Windows Server | 2019/2022 | 需使用WSL2+Docker Desktop |

> 本文档以 **Ubuntu Server 22.04 LTS** 为例进行说明。

### 1.3 端口规划

| 端口 | 服务 | 说明 |
|------|------|------|
| 80 | Nginx | HTTP入口（自动跳转HTTPS） |
| 443 | Nginx | HTTPS入口 |
| 3001 | NestJS API | 后端服务（仅本机访问） |
| 5432 | PostgreSQL | 数据库（仅本机访问） |
| 6379 | Redis | 缓存（仅本机访问） |
| 9000 | MinIO API | 对象存储API（仅本机访问） |
| 9001 | MinIO Console | 对象存储管理面板（可选开放） |
| 9090 | Prometheus | 监控采集（可选开放） |
| 3000 | Grafana | 监控面板（可选开放） |

## 二、依赖安装

### 2.1 系统基础更新

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl wget vim git unzip nginx certbot python3-certbot-nginx
```

### 2.2 安装 Docker

```bash
# 添加Docker官方GPG密钥和仓库
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装Docker引擎
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 验证安装
sudo docker --version
sudo docker compose version

# 将当前用户加入docker组（需重新登录生效）
sudo usermod -aG docker $USER
```

> 加入docker组后，需重新登录SSH会话或使用 `newgrp docker` 生效。

### 2.3 安装 Docker Compose（独立版备用）

```bash
# Docker Compose v2 已作为插件安装，如需独立版可额外安装
sudo curl -L "https://github.com/docker/compose/releases/download/v2.24.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
docker-compose --version
```

### 2.4 Nginx 配置

```bash
# 确认Nginx版本
nginx -v

# 启动Nginx并设置开机自启
sudo systemctl enable nginx
sudo systemctl start nginx
```

## 三、域名配置和DNS

### 3.1 域名要求

- 主域名：`crm.example.com`（请替换为实际域名）
- 建议同时配置：`www.crm.example.com` 指向同一服务器

### 3.2 DNS 记录配置

| 类型 | 主机记录 | 记录值 | TTL |
|------|----------|--------|-----|
| A | `@` | `服务器公网IP` | 600 |
| A | `www` | `服务器公网IP` | 600 |

> DNS 生效通常需要 5-30 分钟，可使用 `dig crm.example.com` 或 `nslookup crm.example.com` 验证。

### 3.3 验证域名解析

```bash
# 在服务器上验证
curl -I http://crm.example.com
# 预期返回 200 OK 或 301/302 跳转
```

## 四、HTTPS证书申请（Let's Encrypt）

### 4.1 使用 Certbot 自动申请

```bash
# 确保Nginx已配置默认站点监听80端口
sudo certbot --nginx -d crm.example.com -d www.crm.example.com
```

按提示操作：
1. 输入邮箱地址（用于续期提醒）
2. 同意服务条款
3. 选择是否订阅邮件
4. Certbot 会自动修改 Nginx 配置并添加 HTTPS 跳转

### 4.2 验证 HTTPS

```bash
curl -I https://crm.example.com
# 预期返回 200 OK，且证书有效
```

### 4.3 自动续期测试

```bash
# 测试自动续期（模拟运行，不实际续期）
sudo certbot renew --dry-run

# 查看定时任务
sudo systemctl status certbot.timer
```

> Let's Encrypt 证书有效期为 90 天，Certbot 已自动配置 systemd timer 续期。

### 4.4 手动续期（备用）

```bash
sudo certbot renew --force-renewal
sudo systemctl reload nginx
```

## 五、环境变量配置

### 5.1 创建项目目录

```bash
sudo mkdir -p /opt/crm-platform
cd /opt/crm-platform
sudo chown $USER:$USER /opt/crm-platform
```

### 5.2 生产环境 .env 模板

在 `/opt/crm-platform/.env` 创建以下文件：

```bash
# ==========================================
# 宁夏CRM作战地图 — 生产环境配置
# ==========================================

# --- 应用基础 ---
NODE_ENV=production
APP_NAME=宁夏CRM作战地图
APP_VERSION=1.0.0

# --- 后端服务 ---
BACKEND_PORT=3001
BACKEND_HOST=0.0.0.0
API_PREFIX=/api

# --- 数据库 (PostgreSQL) ---
DATABASE_URL=postgresql://crm_user:YOUR_STRONG_PASSWORD@postgres:5432/crm_production?schema=public
POSTGRES_USER=crm_user
POSTGRES_PASSWORD=YOUR_STRONG_PASSWORD
POSTGRES_DB=crm_production
POSTGRES_PORT=5432

# --- 缓存 (Redis) ---
REDIS_URL=redis://redis:6379/0
REDIS_PASSWORD=YOUR_REDIS_PASSWORD

# --- 文件存储 (MinIO) ---
MINIO_ENDPOINT=minio
MINIO_PORT=9000
MINIO_CONSOLE_PORT=9001
MINIO_ROOT_USER=crmadmin
MINIO_ROOT_PASSWORD=YOUR_MINIO_PASSWORD
MINIO_BUCKET=crm-files
MINIO_USE_SSL=false
MINIO_REGION=cn-north-1

# --- JWT 认证 ---
JWT_SECRET=YOUR_32_CHAR_JWT_SECRET_KEY_HERE_123
JWT_EXPIRES_IN=7d
JWT_REFRESH_EXPIRES_IN=30d

# --- 日志 ---
LOG_LEVEL=info
LOG_MAX_FILES=30
LOG_MAX_SIZE=50m

# --- 跨域 (生产环境前端域名) ---
CORS_ORIGIN=https://crm.example.com

# --- 邮件通知 (SMTP) ---
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=noreply@example.com
SMTP_PASS=YOUR_SMTP_PASSWORD
SMTP_FROM="宁夏CRM系统" <noreply@example.com>

# --- 监控与告警 ---
ALERT_WEBHOOK_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY
ENABLE_PROMETHEUS=true
METRICS_PORT=9091
```

> 请将 `YOUR_xxx` 占位符替换为实际的安全密码和密钥。密码建议使用 `openssl rand -base64 32` 生成。

### 5.3 生成强密码

```bash
# 生成数据库密码
openssl rand -base64 24

# 生成 JWT 密钥（32位以上）
openssl rand -base64 48

# 生成 Redis 密码
openssl rand -base64 16

# 生成 MinIO 密码（至少8位）
openssl rand -base64 12
```

## 六、部署步骤

### 6.1 创建 docker-compose.prod.yml

```bash
cat > /opt/crm-platform/docker-compose.prod.yml << 'EOF'
version: '3.8'

services:
  postgres:
    image: postgres:16-alpine
    container_name: crm-postgres
    restart: always
    environment:
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB: ${POSTGRES_DB}
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./backups:/backups
    ports:
      - "127.0.0.1:5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - crm-network

  redis:
    image: redis:7-alpine
    container_name: crm-redis
    restart: always
    command: redis-server --requirepass ${REDIS_PASSWORD}
    volumes:
      - redis_data:/data
    ports:
      - "127.0.0.1:6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - crm-network

  minio:
    image: minio/minio:latest
    container_name: crm-minio
    restart: always
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: ${MINIO_ROOT_USER}
      MINIO_ROOT_PASSWORD: ${MINIO_ROOT_PASSWORD}
    volumes:
      - minio_data:/data
    ports:
      - "127.0.0.1:9000:9000"
      - "127.0.0.1:9001:9001"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
      interval: 30s
      timeout: 10s
      retries: 3
    networks:
      - crm-network

  backend:
    image: crm-backend:latest
    container_name: crm-backend
    restart: always
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
      minio:
        condition: service_healthy
    env_file:
      - .env
    environment:
      - DATABASE_URL=postgresql://${POSTGRES_USER}:${POSTGRES_PASSWORD}@postgres:5432/${POSTGRES_DB}?schema=public
      - REDIS_URL=redis://:${REDIS_PASSWORD}@redis:6379/0
    ports:
      - "127.0.0.1:3001:3001"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3001/api/health"]
      interval: 30s
      timeout: 10s
      retries: 3
    networks:
      - crm-network

  frontend:
    image: crm-frontend:latest
    container_name: crm-frontend
    restart: always
    depends_on:
      - backend
    ports:
      - "127.0.0.1:3000:80"
    networks:
      - crm-network

volumes:
  postgres_data:
    driver: local
  redis_data:
    driver: local
  minio_data:
    driver: local

networks:
  crm-network:
    driver: bridge
EOF
```

### 6.2 构建和启动服务

```bash
cd /opt/crm-platform

# 1. 拉取/构建镜像
# 后端镜像构建（从CI/CD流水线推送或本地构建）
docker pull your-registry/crm-backend:latest
docker tag your-registry/crm-backend:latest crm-backend:latest

# 前端镜像构建
docker pull your-registry/crm-frontend:latest
docker tag your-registry/crm-frontend:latest crm-frontend:latest

# 2. 启动所有服务
sudo docker compose -f docker-compose.prod.yml up -d

# 3. 查看启动状态
sudo docker compose -f docker-compose.prod.yml ps

# 4. 查看实时日志
sudo docker compose -f docker-compose.prod.yml logs -f
```

### 6.3 配置 Nginx 反向代理

创建 `/etc/nginx/sites-available/crm`：

```bash
sudo tee /etc/nginx/sites-available/crm << 'EOF'
server {
    listen 80;
    server_name crm.example.com www.crm.example.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name crm.example.com www.crm.example.com;

    ssl_certificate /etc/letsencrypt/live/crm.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/crm.example.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 前端静态资源
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:3001/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_connect_timeout 300s;
    }

    # WebSocket 支持（实时通知）
    location /notifications {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }

    # 上传文件大小限制
    client_max_body_size 100M;
}
EOF

# 启用站点
sudo ln -s /etc/nginx/sites-available/crm /etc/nginx/sites-enabled/crm
sudo nginx -t
sudo systemctl reload nginx
```

> 请将 `crm.example.com` 替换为实际域名。

## 九、回滚方案

### 9.1 备份恢复回滚

如部署失败，使用最新备份恢复：

```bash
# 1. 停止当前服务
sudo docker compose -f docker-compose.prod.yml down

# 2. 恢复数据卷（如有数据卷备份）
# sudo tar -xzf /backups/postgres_data_YYYYMMDD.tar.gz -C /var/lib/docker/volumes/

# 3. 恢复数据库
sudo docker exec -i crm-postgres psql -U crm_user -d crm_production < /backups/db_backup_YYYYMMDD.sql

# 4. 回退到上一版本镜像
sudo docker tag crm-backend:previous crm-backend:latest
sudo docker tag crm-frontend:previous crm-frontend:latest

# 5. 重新启动
sudo docker compose -f docker-compose.prod.yml up -d
```

### 9.2 快速回滚脚本

创建 `/opt/crm-platform/rollback.sh`：

```bash
#!/bin/bash
set -e

BACKUP_DIR="/backups"
DATE=$(date +%Y%m%d_%H%M%S)
LOG_FILE="/var/log/crm-rollback-${DATE}.log"

echo "[$(date)] 开始回滚..." | tee -a $LOG_FILE

# 停止服务
sudo docker compose -f docker-compose.prod.yml down 2>&1 | tee -a $LOG_FILE

# 恢复数据库（使用最新备份）
LATEST_DB=$(ls -t ${BACKUP_DIR}/db_backup_*.sql | head -1)
echo "[$(date)] 恢复数据库: ${LATEST_DB}" | tee -a $LOG_FILE
sudo docker exec -i crm-postgres psql -U crm_user -d crm_production < ${LATEST_DB} 2>&1 | tee -a $LOG_FILE

# 启动服务
sudo docker compose -f docker-compose.prod.yml up -d 2>&1 | tee -a $LOG_FILE

echo "[$(date)] 回滚完成" | tee -a $LOG_FILE
```

```bash
chmod +x /opt/crm-platform/rollback.sh
```

### 9.3 蓝绿部署（高级）

对于零停机更新，可采用蓝绿部署：

```bash
# 1. 启动绿色环境（新版本）
sudo docker compose -f docker-compose.prod.yml -p crm-green up -d

# 2. 验证绿色环境
# ... 执行验证步骤 ...

# 3. 切换 Nginx 到绿色环境
sudo sed -i 's/3000/3002/g' /etc/nginx/sites-available/crm
sudo systemctl reload nginx

# 4. 停止蓝色环境（旧版本）
sudo docker compose -f docker-compose.prod.yml -p crm-blue down
```
