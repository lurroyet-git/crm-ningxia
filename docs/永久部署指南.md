## 前置准备

### 1. 创建 GitHub 仓库

```bash
# 在 GitHub 上创建新仓库（例如：crm-ningxia）
# 然后执行：
git remote add origin https://github.com/你的用户名/crm-ningxia.git
git branch -M main
git push -u origin main
```

### 2. 配置环境变量

在部署平台上设置以下环境变量：

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `DATABASE_URL` | Neon PostgreSQL 连接字符串 | `postgresql://user:pass@host/db?sslmode=require` |
| `JWT_SECRET` | JWT 签名密钥 | 32位以上随机字符串 |
| `JWT_EXPIRES_IN` | Token 有效期 | `7d` |
| `PORT` | 服务端口 | `3001` |
| `BCRYPT_SALT_ROUNDS` | 密码哈希强度 | `10` |

## 方案一：Render.com（推荐）

**免费额度**：Web Service 750小时/月，PostgreSQL 免费

### 步骤

1. **注册 Render**：https://render.com（可用 GitHub 账号登录）

2. **创建 Blueprint**
   - 在 Dashboard 点击 "New +" → "Blueprint"
   - 选择 `crm-ningxia` GitHub 仓库
   - Render 会自动读取 `render.yaml` 配置

3. **配置环境变量**
   - 在 Render 面板找到 `crm-ningxia-api` 服务
   - 进入 Environment → 添加 `DATABASE_URL` 和 `JWT_SECRET`

4. **等待部署完成**
   - 后端：https://crm-ningxia-api.onrender.com
   - 前端：https://crm-ningxia-web.onrender.com

### 更新代码

```bash
git push origin main
# Render 会自动重新部署
```

## 方案二：Railway.app

**免费额度**：$5/月（足够运行本项目）

### 步骤

1. **注册 Railway**：https://railway.app（可用 GitHub 账号登录）

2. **创建项目**
   - 点击 "New Project" → "Deploy from GitHub repo"
   - 选择 `crm-ningxia` 仓库

3. **添加 PostgreSQL**
   - 点击 "New" → "Database" → "Add PostgreSQL"
   - Railway 会自动生成 `DATABASE_URL`

4. **配置环境变量**
   - 进入项目 Settings → Variables
   - 添加 `JWT_SECRET`（随机字符串）

5. **部署**
   - Railway 自动读取 `railway.toml` 和 `Dockerfile`
   - 访问分配的域名即可

## 方案三：自有服务器

### 服务器要求

| 配置 | 最低 | 推荐 |
|------|------|------|
| CPU | 1核 | 2核 |
| 内存 | 1GB | 2GB |
| 磁盘 | 10GB SSD | 20GB SSD |
| 带宽 | 1Mbps | 5Mbps |
| 系统 | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS |

### 一键部署脚本

```bash
# 1. 克隆代码
git clone https://github.com/你的用户名/crm-ningxia.git
cd crm-ningxia

# 2. 配置环境变量
cp backend/.env.template backend/.env
# 编辑 backend/.env 填入 DATABASE_URL 和 JWT_SECRET

# 3. 运行部署脚本
chmod +x deploy.sh
./deploy.sh production
```

### 手动部署步骤

```bash
# 安装 Docker 和 Docker Compose
sudo apt update && sudo apt install -y docker.io docker-compose

# 克隆代码
git clone https://github.com/你的用户名/crm-ningxia.git
cd crm-ningxia

# 构建并启动
docker-compose -f docker-compose.prod.yml up -d

# 数据库迁移
docker exec crm-backend npx prisma migrate deploy

# 验证
curl http://localhost:3001/health
```

### 配置 Nginx + HTTPS

```bash
# 安装 Nginx 和 Certbot
sudo apt install -y nginx certbot python3-certbot-nginx

# 复制配置
sudo cp nginx/nginx.conf /etc/nginx/sites-available/crm
sudo ln -s /etc/nginx/sites-available/crm /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl restart nginx

# 申请 SSL 证书
sudo certbot --nginx -d crm.yourdomain.com
```

## 数据库迁移

### 从 Neon 迁移到自有 PostgreSQL

```bash
# 1. 导出 Neon 数据
pg_dump $DATABASE_URL > backup.sql

# 2. 导入到新数据库
psql $NEW_DATABASE_URL < backup.sql

# 3. 更新环境变量
# 修改 DATABASE_URL 为新数据库地址
```

## 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 后端启动失败 | DATABASE_URL 错误 | 检查数据库连接字符串 |
| 前端白屏 | API 地址错误 | 检查 VITE_API_BASE_URL |
| 登录失败 | JWT_SECRET 不一致 | 确保前后端使用相同密钥 |
| 数据库连接超时 | Neon 免费版休眠 | 首次访问可能慢，后续正常 |
| 文件上传失败 | MinIO 未配置 | 可选，不影响核心功能 |
